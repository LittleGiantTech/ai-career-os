## Project Mission

AI Career OS helps aspiring AI Engineers and AI Architects stay focused, learn efficiently, and avoid information overload.

The application is NOT a generic task manager.

The application is a learning operating system.

---

## Core Rules

Rule 1:
Only one active sprint.

Rule 2:
Everything else goes into the Parking Lot.

Rule 3:
Projects validate learning.

Rule 4:
Consistency beats intensity.

Rule 5:
The application should always answer:

"What should I do next?"

---

## Development Rules

When building features:

1. Simplicity over complexity.
2. Mobile-first design.
3. Vercel deployment compatibility.
4. TypeScript everywhere.
5. Reusable components.
6. Use shadcn/ui components.
7. Follow Next.js best practices.

---

## AI Behavior Rules

The AI should never:

- Encourage shiny-object syndrome.
- Suggest abandoning current sprint.
- Recommend ten different topics simultaneously.

The AI should always:

- Reinforce current sprint.
- Store future interests in Parking Lot.
- Recommend project-based learning.
- Explain WHY a topic matters.

---

## Every Task

Before implementing:

1. Explain goal.
2. Explain architecture impact.
3. Explain files affected.
4. Implement solution.
5. Explain testing steps.

Always work incrementally.
Never rewrite working systems unnecessarily.
